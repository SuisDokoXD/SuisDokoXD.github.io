第 23 周 devlog

发表于2026-06-06|更新于2026-06-06|项目记录
|总字数:520|阅读时长:1分钟|浏览量:

本周概览

inkforge 这周一口气往前推了不少，主要是两块：skill 引擎从测试到交互链路补了一轮，UI 层在做动效和主题的统一。其他三个项目这周没动，精力都集中在 inkforge 上了。节奏上还算稳，没有特别鬼打墙的坑。

各项目进展

inkforge

  • skill 引擎这边把 template/trigger-scheduler/validate 几个模块的单测都补完了，顺手接上了 turbo test 跑 CI 流程。然后删掉了 runner 里无引用的老代码，校正了模板宏的注释，整体结构干净了一些。
  • 交互链路上，执行 prompt 现在能按 UI 语言做本地化，SkillPage 的模板也支持了实时预览。还新增了 verify-skill 和 swap-sqlite-abi 两个脚本，做工具链上的补全。
  • UI 这边改动不小：对话框、吐司、列表全部接上了 AnimatedDialog 和 motion 动效，缩放时 backdrop-blur 会挂起避免掉帧。ActivityBar 换上了 lucide 的单色图标，加了空状态引导页面，浅色主题也做了适配。
  • 配色上从之前的 amber 统一换成了系统蓝 accent，渲染层默认主题改为 light。桌宠那边把 4 个物种的 SVG 收敛成了单一 Lottie 吉祥物。
  • 最后补了一次清理死代码的提交。

卡点与思考

这次没什么特别卡的，UI 动效和主题切换的适配比想象中要顺畅。唯一值得记一下的是：把几个 SVG 物种合并成 Lottie 时，发现不同物种的动画帧率不太一致，最后统一用了 30fps 的导出设置，然后靠脚本动态调整播放速度来模拟不同步伐频率。算是个小 trick 吧。

下周计划

  • 进一步打磨 skill 引擎的异常处理和错误提示，把本地化覆盖到所有用户可见文案
  • UI 层开搞设置页面和插件管理，目前这两块还是空白
  • 看看什么时候能给 inkforge 跑一个完整的端到端流程 demo
